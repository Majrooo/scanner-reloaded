# Rules for Code Modification

- **Make only local, partial changes (diffs).** When adding new features that affect multiple files, do not rewrite entire files. Only add or modify the necessary code blocks.
- **Preserve file structure and formatting.** Never alter the overall structure, formatting, or existing blank lines of a file unless it's the specific goal of the request.
- **Focus on the requested task.** Do not perform unsolicited refactoring or code cleanup. Stick strictly to implementing the requested feature or fix.
- **Keep documentation in sync.** If you add a new feature, update the relevant documentation (e.g., `README.md`) to reflect the changes.
- **Use English for all comments.** All code comments, documentation, and commit messages should be written in English to ensure the project is accessible to a wider audience.
- **Write clear commit messages.** Follow a conventional commit message format (e.g., `feat: Add new feature X`, `fix: Correct bug in Y`).

These rules ensure that collaboration remains efficient, and the codebase stays clean, consistent, and easy to navigate.
